# OpenHealth - карта возможностей

> Сгенерировано из `ui/web/assets/registry.json`. Не править руками - правки затрутся.
> Чтобы обновить после изменения реестра, запустить: `python3 ui/web/gen_capabilities.py`.

Это всегда актуальный, агентно-читаемый индекс того, что дашборды OpenHealth уже умеют сегодня: скины, темы, разделы и метрики с провенансом. Источник правды - реестр; оба скина (V1, V2) рендерятся из него с паритетом. Как расширять систему - см. `EXTENDING.md`.

Версия реестра: 1.

## Скины

Скин - это раскладка и навигация поверх общего движка. Каждый перечисленный скин рендерится из реестра и поддерживает свои темы.

| id | Название | Файл | Темы |
| --- | --- | --- | --- |
| `v1` | Классический (тёмный) | `dashboard.html` | dark, light, brutalist |
| `v2` | Bento (светлый) | `dashboard-v2.html` | bento |

## Метрики

Каждая метрика определяется один раз в реестре и автоматически доступна обоим скинам. Значения приходят из `data.local.json` по id метрики (или `data_key`); при отсутствии данных показывается демо из реестра.

### `recovery` - Восстановление

- id: `recovery`
- Название (ru): Восстановление
- Единица: %
- Источник: whoop
- Тип графика: ring
- Раздел: today
- protocol_ref: recovery@v3
- Провенанс:
  - что: Готовность организма к нагрузке, 0-100%.
  - как: recovery@v3 из HRV (rMSSD, ln-SD), пульса покоя, частоты дыхания и качества сна относительно личной базы.
  - зачем: Высокий recovery - можно грузиться; низкий - приоритет восстановлению.
