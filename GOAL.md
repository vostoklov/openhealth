# GOAL — доделать OpenHealth качественно по всем фронтам

Единый бриф для Claude Code. Консолидирует все комментарии и планы Ильи (сессии 2026-06).
Цель: довести OpenHealth до состояния, в котором он живое доказательство тезиса c1 — здоровье решает образ жизни, а данные без действия бесполезны.

## Видение (стержень, не терять)
- Ядро = ежедневный журнал образа жизни (минимальный friction) ↔ recovery/strain → ПЕРСОНАЛЬНЫЕ корреляции → 1-3 конкретных действия на сегодня. Action-first, не датадрочерство.
- **Агентность — главное.** Дашборд не просто выводит данные. Он цепляется к Claude Code / Codex и ЗАПУСКАЕТ ИИ под локальными данными человека: сгенерировать инсайт, прогнать корреляции, дип-ресёрч по параметру, разобрать транскрипт. Каждая «умная» панель = действие, дёргающее агента/CLI, а не статичный текст. Это ключевая составляющая, без неё продукт мёртвый.
- Local-first: данные и ключи не уходят с устройства. Evidence C1-C5 (C5 сильнейшее, established/RCT; C3 и ниже = вопрос; личный паттерн ≤ C2 пока нет n-of-1). Не врач, red-flags → к клиницисту.
- У каждого своя библиотека deep-research/источников, на которую опирается C1-C5 (knowledge-enrichment слой из c1-презентации).

## Текущее состояние (что есть, на чём строить)
- Движок `openhealth/` (Python stdlib): 10 модулей (pulse, sleep, cycle, body, metabolic, skin, journal, recovery, correlations, travel), коннекторы (apple_health, whoop, google_calendar, oura, garmin), `clinical_optima.py`, `circadian.py`, `evidence.py` (канон C1-C5), `scheduler.py`, `mcp_server.py`, скиллы `.claude/skills/openhealth/` + guardrails. 160 тестов зелёные. Запушено в main (github.com/igindin/openhealth).
- Дашборд `ui/web/dashboard.html` (=index.html) — single-file premium тёмный (Flenteey), GSAP, темы. Экраны: Сегодня, Пульс, Biomarkers, WHOOP Overview/Correlations, Журнал, Тренды, Синхронизация. DEMO-данные (hardcoded DATA), НЕ реальные.
- iOS `ui/ios/` — journal-first, тёмная тема, BUILD SUCCEEDED.
- Данные Ильи для теста: реальные WHOOP-экспорт (12.02-14.03), lab PDF, Atlas/ДНК, HRV deep-research — лежат в его health-os overlay (`~/health-os/`). Нужно подтянуть (см. фронт 3).

## ОБЪЁМ РАБОТ ПО ФРОНТАМ

### 1. Агентность (приоритет, стержень)
- Подключить дашборд к Claude Code / Codex / openhealth CLI. Кнопки «Сгенерировать инсайт», «Прогнать корреляции», «Дип-ресёрч по HRV», «Разобрать транскрипт» реально запускают агента под локальными данными и показывают результат. Не fake.
- Тренды: текстовый insight, который ГЕНЕРИТСЯ агентом по данным (сейчас только графики).
- Архитектура моста: дашборд → локальный endpoint/CLI (`openhealth ...`) → агент → результат в UI.

### 2. Реальные данные вместо demo (после деплоя)
- Мост `openhealth show-summary` (и др.) → JSON → дашборд читает реальные значения, не hardcoded DATA.
- Импортировать РЕАЛЬНЫЕ данные Ильи из `~/health-os/`: WHOOP API/экспорт, lab-анализы, ДНК. Прогнать через движок (import → recovery → корреляции), показать в дашборде как тест на живых данных.

### 3. Реальные подключения (не fake-кнопки) + разделить «загрузки» vs «синхронизации»
- Сейчас раздел «Синхронизация» смешивает разное. Helix/Лаборатории и ДНК — это ЗАГРУЗКА файлов (PDF, raw), а не синхронизация. Развести: «Синхронизации» (живой поток: WHOOP, Apple Health, Oura, Garmin) и «Загрузки» (разовый импорт: lab PDF, ДНК raw, скриншоты).
- Для КАЖДОГО источника продумать реальные шаги: WHOOP = OAuth (реальное подключение, не фиктивное), Apple Health = инструкция экспорта + импорт, Oura/Garmin = OAuth/API, Helix = загрузка PDF → парсинг, ДНК = загрузка raw → разбор. Каждая кнопка ведёт к реальному флоу.

### 4. Синхронизация чек-ина + настройка
- Чеклист «Что сделано сегодня» (главный) и отметки в Журнале = ОДИН компонент, единое состояние, единый вид (взять вид с «Сегодня»).
- Одно место настройки привычек (что трекать), общее для обоих.
- Утренний + вечерний чек-ин по шагам (UX как Sunsama / Mihaly: план дня утром, закрытие дня вечером). Посмотреть в Mobbin/live.

### 5. IA: группировка + скрытие + device-agnostic
- Сгруппировать разделы (их много) в логичные группы.
- Дать скрывать лишние разделы в интерфейсе (настройка видимости).
- Device-agnostic: «WHOOP Overview/Correlations» → «Обзор / Корреляции» (любой трекер: Oura/Garmin/Apple/WHOOP). Источник — переменная, не зашит WHOOP.
- Недоступные разделы (Timeline, Состав тела, Тренировки, Protocols, Research, Отчёты, Дайджесты): дать контент или честный empty-state, не серый disabled молча.

### 6. Новые разделы
- **Ментальное здоровье / эмоции**: daily-отметка эмоционального состояния + отметка эмоций → анализ динамики эмоций. Опционально: анализ транскриптов звонков (если подключить модуль fathom/zoom) → эмоциональные паттерны.
- **Циркадные ритмы** (движок `circadian.py` есть — вывести в UI: окно сна, energy peak/dip, мелатонин, свет).
- **Гормональный фон** (раздел: маркеры, динамика; связь с cycle-модулем).
- **Погода** (влияние на recovery, контекст — как travel/location).
- **Журнал питания**: фото блюд + расписание приёмов.
- **Журнал тренировок**.
- **Skincare-библиотека**: свои продукты + состав (product_analyzer: фото → состав → совет).
- **Библиотека deep-research / источников**: локальная, у каждого своя. Просмотр/добавление своих ресёрчей и пруфов, на которые опирается C1-C5.
- **Регулярные опросники здоровья** (periodic questionnaires: самочувствие, симптомы, скрининги).
