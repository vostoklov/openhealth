# HRV (вариабельность сердечного ритма)
> algo_version: n/a (модуль pulse; readiness-чтение v2 по ln-SD) · источник данных: движок + WHOOP-сырое · редактируемость: параметры в коде

## Что это

Главный личный маркер готовности. Основная метрика — **rMSSD** (root mean square of successive differences, мс): корень из среднего квадрата разностей соседних RR-интервалов. Отражает парасимпатический тонус. На дашборде показывается ночной rMSSD WHOOP (сырое значение из выгрузки); модуль `pulse` умеет считать rMSSD самостоятельно из RR-интервалов (Task Force 1996, golden-tested).

## Формула / алгоритм

**Time-domain (точные формулы, `openhealth/modules/pulse.py`):**

- очистка RR: остаются только интервалы 300-2000 мс (30-200 уд/мин), артефакты выбрасываются;
- `rMSSD = sqrt( mean( (RR[i+1] − RR[i])² ) )`;
- также: SDNN (SD интервалов, ddof=1), pNN50 (% соседних разностей > 50 мс), mean HR = 60000 / mean RR.

**Почему ln.** rMSSD логнормально распределён (длинный правый хвост): одинаковый сдвиг в мс при низком и высоком HRV — разные события. На ln-шкале сдвиги сопоставимы, а «нормальный диапазон» строится честно: baseline ln ± k·SD личного ln(rMSSD) вместо магических порогов 0.9/0.7 (конвенция Altini / HRV4Training, smallest-worthwhile-change).

**Ночная агрегация.** Движок потребляет уже агрегированный провайдером ночной rMSSD как есть (одно значение за ночь). Точное окно агрегации WHOOP («глубокий сон» vs средневзвешенное за ночь) публично неоднозначно — наша ln/SD-математика применяется к скорингу и baseline, не к переагрегации сырых RR.

**Baseline и SWC.** Baseline = геометрическое среднее `exp(mean(ln rMSSD))` за 28 дней (см. recovery.md). Личная «полоса нормы» (smallest worthwhile change): `z = (ln(сегодня) − ln(baseline)) / ln_SD`:

- `z >= −1` — обычный день;
- `−2 <= z < −1` — ниже обычного диапазона, стоит отметить;
- `z < −2` — заметно вне диапазона (частое после плохого сна, алкоголя, болезни).

## Источники и доверие

- Определения метрик — Task Force of ESC/NASPE (1996).
- Ln-скоринг и личные SD-полосы — Altini / HRV4Training.
- Однодневное readiness-чтение — личный паттерн, cap C2 (вопрос, не вердикт), формулируется с открытыми вопросами «как спал, был ли алкоголь».

## Известные ограничения

- Frequency-domain (LF/HF) считается наивным DFT по линейно интерполированному тахограмму — годится для тренда, явно низкое доверие (запланирована замена на Welch).
- Ночной rMSSD от провайдера не переагрегируется — наследуем его методологию вслепую.
- HRV чувствителен ко всему сразу (сон, алкоголь, стресс, болезнь, тренировки) — одно число не объясняет причину.
