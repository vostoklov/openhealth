Health Sprint · OpenHealth

Твоя система здоровья — старт за 15 минут

Ты управляешь всем через Claude Code или Codex. Говоришь — он логирует твои данные, находит паттерны, помогает собрать твою систему. Учить интерфейс не нужно. Данные остаются на твоей машине.

Не нужно 20 лет анализов или WHOOP. Старт = айфон и один вопрос. Никого не догоняем. Те, кто грузит десятилетия PDF, — это вдохновение, а не планка.

Как это устроено — что где

ДанныеЧто есть про тебяApple Health, анализы, заметки о сне/энергии, календарь
↓
СкилыОдин скил = одна областьсон, анализы, цикл, восстановление, трекинг
↓
АгентыДумают за тебя, осторожноdeep-research цепочкой + консилиум: гипотеза / критик / безопасность
↓
СистемаТвой результатToday, инсайты с уровнем доверия, ежедневные действия

Данные → скилы → агенты → твоя персональная система. Слои отдельные — поэтому система честная, а не «всё в одной куче».

Четыре шага

  1. Запусти setup
    Открой папку в Claude Code и скажи: «run make setup».
  2. Скажи /kit и что хочешь
    Например: «сон и ежедневный чек-ин». Поставит только нужное, без перегруза.
  3. Заведи один источник данных
    iPhone → Здоровье → твоё фото → «Экспортировать данные» → .zip. Скажи: «импортируй мой Apple Health export». Или просто один анализ. Или ничего — начнём с того, что есть.
  4. Выбери один вопрос + /checkin
    «Что ломает мой сон?» / «Почему по средам нет сил?». Раз в день — короткий чек-ин. Всё.

Что внутри

  • СигналыApple Health, WHOOP, CSV, анализы-PDF
  • Знаниясон, анализы, цикл, тело, метаболизм (HRV — опционально)
  • Действия/track привычки, /checkin, /protocol (n-of-1)
  • Довериеdeep-research + консилиум, оценка C1–C5

Три правила

  • Помощник для размышления, не врач. Без диагнозов. Решения — с врачом. Тревожное → стоп и к специалисту.
  • У каждого вывода есть уровень доверия (C1–C5). Низкий — формулируется как вопрос, а не факт.
  • Не копируй чужой скил вслепую. Репозиторий может быть сложнее, чем нужно, или под другую цель. Адаптируй под свою.

Застрял или рябит в глазах? Это нормально. Сделай только шаг 1 и один вопрос — остальное на следующих встречах. Спроси в чате спринта или у поддержки.

Local-first: ничего не загружается в облако. Реальные данные и ключи — только на твоей машине. OpenHealth ещё сыроват местами — это нормально, ты собираешь свою систему.
